# Verbal Identity

This document defines how Obsura should sound and which product language should
repeat consistently across repositories and public surfaces.

See also: [Tone Of Voice](tone-of-voice.md), [Messaging](messaging.md),
[Naming](naming.md)

## Core Messaging Style

Obsura should speak in a way that is:

- clear
- concise
- exact
- calm
- modern
- trustworthy
- practical

The writing should feel like a serious product team explaining a real tool, not
like a campaign trying to create excitement without substance.

## Preferred Language

Prefer these terms because they match the standards baseline and the product's
actual direction:

- review
- redact
- secure export
- styling
- visual replacement
- self-hosted
- workflow
- screenshots
- technical text
- safe sharing
- platform
- workbench
- recognizer
- pack
- plugin

## Terms To Avoid Or Use Carefully

Avoid these unless the distinction is deliberate and documented:

- censor
- anonymizer
- scrubber
- surveillance
- spy
- classified
- stealth
- military-grade
- zero-friction AI magic
- fully autonomous redaction

Use `masking` only when it refers to a specific UI style or technical behavior,
not as a loose replacement for `redaction`.

## Product Wording Guidance

Prefer wording that describes the actual sequence of work:

- detect
- review
- adjust
- style
- replace
- export
- share safely

Avoid wording that skips review or implies hidden automation:

- instantly fixes everything
- catches all secrets automatically
- guarantees perfect privacy by default

## Phrasing Patterns

Good patterns:

- "Review and redact before you share."
- "Safely share screenshots, logs, and technical text."
- "Detect, review, style, and securely remove sensitive information."
- "Built for fast daily-use redaction workflows."

Weak patterns:

- "Reinvent your privacy posture with AI-native workflows."
- "A revolutionary secrecy platform for the modern enterprise."
- "The ultimate all-in-one content sanitization universe."

## Product Naming Standard

Use `Obsura` for the product name in prose.

Use lowercase `obsura-*` for repository names, package prefixes, and technical
identifiers that follow the repo naming system.
