# Social And Profile Usage

This document covers GitHub avatars, social previews, bios, and similar profile
surfaces.

See also: [Logo Usage](logo-usage.md), [Website And Product Copy](website-and-product-copy.md)

## GitHub Organization Avatar

Use the mark, not the full logo.

Rules:

- keep the mark centered with clear padding
- use a controlled white, paper, or mist background export if platform contrast
  is inconsistent
- do not use tiny wordmarks that become unreadable at avatar size

Because the current checked-in icon set is transparent black only, create a
dedicated avatar export in `assets/previews/github/` before using it on surfaces
where dark-mode contrast would be unreliable.

## GitHub Social Preview

Prefer a simple composition:

- full logo if available, otherwise the mark plus the word `Obsura`
- one short line of factual product description
- restrained paper or mist background with one floating white card or mark tile
- one restrained periwinkle accent if emphasis is needed

Avoid screenshot collages, dense text, or security cliches.

## Social Bio Options

Short:

"Open-source, self-hosted redaction for screenshots, logs, and technical text."

Medium:

"Obsura is an open-source, self-hosted redaction platform for safely sharing
screenshots, text, logs, and technical files."

Compact:

"Review-first redaction for real technical work."

## Website Hero Framing

For website or profile headers, prefer:

- a clear product description
- a workflow-oriented tagline if needed
- a direct explanation of who the product is for
- light, calm framing rather than dark security theater

Do not rely on a slogan alone to explain the product.

## Short Profile Rule

If there is room for only one line, use the one-line description from
[Messaging](messaging.md) before using a tagline by itself.
