# Brand Overview

This document defines what Obsura should feel like before any deeper visual or
copy decisions are made.

See also: [Brand Principles](brand-principles.md),
[Verbal Identity](verbal-identity.md), [Visual Direction](visual-direction.md)

## What Obsura Is As A Brand

Obsura is the brand of an open-source, self-hosted redaction platform built for
people who need to inspect, redact, and share sensitive technical content
safely and quickly.

The brand should make that use case obvious. It should not feel like a generic
security company or a vague AI utility.

## Brand Essence

Obsura is controlled visibility with calm precision.

The product helps users decide what should remain visible, what should be
concealed, and what should be safe to share after review.

## Product Personality

Obsura should feel:

- precise
- calm
- modern
- trustworthy
- clear
- restrained
- technical
- polished
- quietly premium
- privacy-first

Obsura should not feel:

- playful
- theatrical
- surveillance-themed
- fear-based
- legal-only
- over-marketed
- vague about its actual use

## Desired User Perception

The intended reaction is practical rather than emotional:

"This looks like a precise, modern workbench for safely preparing sensitive
technical content to share."

For a user pasting a VPS screenshot, terminal capture, log snippet, or config
file, Obsura should signal:

- fast review
- clear control
- safe export
- modern software quality
- no security theater

## What The Brand Should Signal

Obsura should consistently signal:

- safe sharing instead of blanket secrecy theater
- calm confidence instead of alarm
- review-first workflows instead of opaque automation
- technical competence without jargon overload
- open-source professionalism without corporate stiffness
- visual softness around exact work
- visual restraint without sterility

## Standard

When a brand decision is unclear, choose the option that feels more credible to
someone sanitizing operational content under time pressure.
