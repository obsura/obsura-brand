# Taglines

Taglines are optional. Obsura should still read clearly without one.

Use a tagline only when it improves a surface such as a hero section, social
preview, launch post, or profile header.

See also: [Messaging](messaging.md),
[Website And Product Copy](website-and-product-copy.md)

## Primary Recommended Tagline

`Review. Redact. Share safely.`

Why this is the default:

- it matches the review-first product direction
- it describes a workflow rather than a vague promise
- it fits docs, website, and product-adjacent surfaces

## Secondary Options

`Redact with confidence.`

Use for compact surfaces that need a shorter line with a higher-trust tone.

`Modern redaction for real work.`

Use when the surrounding copy already explains what kind of work Obsura is for.

`Safe sharing for screenshots, logs, and technical text.`

Use when the surface needs a more descriptive, task-oriented line.

## Usage Rule

Do not force a tagline into every README, repo header, or asset. Many surfaces
work better with the one-line description from [Messaging](messaging.md).
