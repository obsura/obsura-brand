# Naming

This document defines the official naming system for the Obsura brand
repository.

See also: [Repo Descriptions](repo-descriptions.md),
[Asset Organization](asset-organization.md)

## Product Name

- Correct: `Obsura`
- Incorrect in prose: `obsura`
- Incorrect unless required by platform constraints: `OBSURA`

Use `Obsura` in documentation, product copy, release notes, and public text.

## Repository Names

- Use lowercase kebab-case
- Use the `obsura-` prefix for top-level repositories
- Choose names by responsibility, not by implementation detail

Examples:

- `obsura-standards`
- `obsura-brand`
- `obsura-web`
- `obsura-api`
- `obsura-engine`
- `obsura-docs`
- `obsura-packs`
- `obsura-windows`

If a top-level `obsura` repository is ever introduced, document that exception
in `obsura-standards` before making it part of the baseline naming system.

## Asset File Names

Use lowercase kebab-case file names with stable, literal descriptors.

Recommended patterns:

- `obsura-logo-primary.svg`
- `obsura-logo-dark.svg`
- `obsura-logo-light.svg`
- `obsura-mark.svg`
- `obsura-mark-dark.svg`
- `obsura-mark-light.svg`
- `obsura-icon-16.png`
- `obsura-icon-96.png`

## Preview And Social Asset Names

Use descriptive names that identify surface and variant clearly.

Examples:

- `obsura-github-avatar-dark.png`
- `obsura-github-social-preview.png`
- `obsura-website-hero-preview-dark.png`
- `obsura-product-header-preview.png`

## Document File Names

- Use lowercase kebab-case for Markdown documents
- Give each document one clear purpose
- Do not create multiple files that define the same rule differently

## Names To Avoid

Do not use:

- `logo new final.svg`
- `obsura latest 2.png`
- `final-final-v3.svg`
- `new-brand-really-final.png`

## Rule

The file name should make the asset's purpose obvious without opening it.
